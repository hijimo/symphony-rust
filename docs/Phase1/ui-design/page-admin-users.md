## 4. 弹窗设计

### 4.1 添加用户弹窗

```
+----------------------------------+
| 添加用户                     [X] |
+----------------------------------+
|                                  |
|  用户名 *                        |
|  +----------------------------+  |
|  | (请输入用户名)              |  |
|  +----------------------------+  |
|                                  |
|  密码 *                          |
|  +----------------------------+  |
|  | (请输入密码)           [o] |  |
|  +----------------------------+  |
|                                  |
|  显示名                          |
|  +----------------------------+  |
|  | (请输入显示名)              |  |
|  +----------------------------+  |
|                                  |
|  角色 *                          |
|  +----------------------------+  |
|  | user                    [v]|  |
|  +----------------------------+  |
|                                  |
|          [取消]    [确认添加]    |
+----------------------------------+
```

| 属性 | 值 |
|------|-----|
| 宽度 | 480px (xs: 全屏) |
| 内边距 | 24px |
| 标题 | h6 "添加用户" |
| 关闭按钮 | 右上角 IconButton (Close) |

#### 表单字段
| 字段 | 类型 | 必填 | 验证规则 |
|------|------|------|----------|
| 用户名 | text | 是 | 3-32字符，仅字母数字下划线 |
| 密码 | password | 是 | 最少6字符 |
| 显示名 | text | 否 | 最多64字符 |
| 角色 | select | 是 | admin / user |

#### 按钮
- 取消: variant="text", color="inherit"
- 确认添加: variant="contained", color="primary"
- Loading 时确认按钮显示 spinner

## 5. 交互状态

### 5.1 Loading 状态
- 表格区域显示 `<Skeleton>` 占位（5行）
- 或使用 `<LinearProgress>` 在表格顶部

### 5.2 空状态
```
+-------------------------------------------------------+
|                                                       |
|              [PeopleOutline 图标, 64px]               |
|              暂无用户数据                              |
|              点击"添加用户"创建第一个用户              |
|                                                       |
+-------------------------------------------------------+
```

### 5.3 搜索无结果
```
+-------------------------------------------------------+
|                                                       |
|              [SearchOff 图标, 48px]                   |
|              未找到匹配的用户                          |
|              请尝试其他搜索关键词                      |
|                                                       |
+-------------------------------------------------------+
```

### 5.4 操作反馈
| 操作 | 反馈 |
|------|------|
| 添加成功 | Snackbar success: "用户创建成功" |
| 重置密码成功 | Snackbar success: "密码重置成功" |
| 删除成功 | Snackbar success: "用户已删除" |
| 操作失败 | Snackbar error: 具体错误信息 |

### 5.5 错误状态
- 网络错误: 表格区域显示错误提示 + 重试按钮
- 权限不足: 跳转到 403 页面或显示提示

## 6. 响应式行为

### Desktop (lg+)
- 完整表格显示所有列
- 侧边栏展开

### Tablet (md)
- 隐藏 ID 列
- 侧边栏收起为图标模式
- 搜索框宽度自适应

### Mobile (xs - sm)
- 表格改为卡片列表模式
- 每个用户一张卡片，包含所有信息
- 操作按钮在卡片底部
- 侧边栏隐藏（汉堡菜单）
- 添加按钮改为 FAB (Floating Action Button)

### 移动端卡片布局
```
+----------------------------+
| [Avatar] admin             |
| 管理员                     |
| 角色: [admin]  ID: 1      |
| 创建: 2024-01-01          |
| [重置密码]  [删除]        |
+----------------------------+
```

## 7. 键盘交互

| 按键 | 行为 |
|------|------|
| Tab | 在工具栏和表格间导航 |
| Enter | 在搜索框中触发搜索 |
| Escape | 关闭弹窗 |
| Delete | 选中行时触发删除确认 |

---

## 8. 无障碍

- 表格使用语义化 `<table>` 标签
- 操作按钮有明确的 `aria-label`（如 "重置用户张三的密码"）
- 弹窗使用 `role="dialog"` 和 `aria-labelledby`
- 分页控件有 `aria-label="分页导航"`
- 搜索框有 `aria-label="搜索用户"`
- 删除确认弹窗聚焦到取消按钮（防止误操作）
